## Backward Incompatibilities {#sec-release-26.11-incompatibilities}

<!-- To avoid merge conflicts, consider adding your item at an arbitrary place in the list instead. -->

- `programs.pqos-wrapper` module has been deleted
  as the corresponding package has been dropped from nixpkgs.

- `programs.benchexec.enable = true`
  no longer sets `programs.pqos-wrapper.enable = true`
  as the corresponding module has been deleted.

- `services.qbittorrent`: when `serverConfig` is non-empty, the module now pre-seeds `BitTorrent/Session/QueueingSystemEnabled = false` to match qBittorrent's documented default. Previously, writing any `serverConfig` caused qBittorrent to misclassify the install as an upgrade from a pre-4.3 version and inject the historical default (`true`), enabling torrent queueing silently. Users of `serverConfig` that currently haven't set `QueueingSystemEnabled` must set `serverConfig.BitTorrent.Session.QueueingSystemEnabled = true` to retain the historical default after upgrading. Users that don't use `serverConfig` or have already set `QueueingSystemEnabled` are not impacted.
